PROBLEM

PROBLEMS TO SOLVE

  • Subpar service quality-Global demand for CDN is exploding, but due to the complex network environment and deployment method, access experience for end users is hard to guarantee.
  • Inefficient use of network resources-Standardized global planning and optimization is highly difficult due to the multitude of CDN providers and scheduling methods coexisting in the market, putting a ceiling to global network usage efficiency.
  • Central node bottleneck-Users have been increasingly demanding better quality in terms of picture, video streaming, which puts higher and higher loads on the network that a centralized service model finds harder and harder to keep up.
  • Global CDN pricing difference is significant-Due to the rising acquisition cost of server resources and different usage models, the prices many CDN service providers quote are high and varied.

Solution

coldCDN Solution

Using token incentivized distributed CDN, we can unlock the potential of all idle server resources and a variety of node types through a standardized open protocol that will significantly improve network usage efficiency and decrease cost

  • A standardized protocol
  • Users can choose to voluntarily contribute resources to the network in exchange for token rewards
  • Decrease the cost of using CDN
Profit generation happens mainly off-chain. Users access and pay for CDN service. The additional cost of running existing terminal nodes is minimal because most of the investment and operating cost of existing servers that have idle capacity is sunk cost. Terminal nodes are incentivized by token rewards to voluntarily contribute resources to the network. The effectiveness of the network increases exponentially with the increase in the number of network nodes. Consequently, end users will be willing to pay a higher fee for the CDN service that is still very competitively priced compare with other centralized CDN providers. We take the revenue earned to repurchase and burn outstanding tokens to align the interests of every ecosystem participant and the interest of the protocol as a whole.

F.A.Q

Frequently Asked Questions

  • How to prevent the scenario where terminal users used IP pools to fake a large amount of access requests and maliciously earn tokens

    We would obfuscate file names and hide the original urls. We also use IP statistical analysis to screen out cheating terminal nodes.

  • What would happen if users used coldCDN to cache illegal content?

    We could use AI systems to discover and delete illegal content.

  • CDN companies in China need to apply for licenses, will this be a problem for coldCDN?

    Decentralized and distributed nodes do not have such problems.

  • What is the safety assurance for enterprise level users?

    The code is open source so any users can check and verify the security of the protocol.

  • How to deal with terminal nodes that cheat and do not transmit data?

    All terminal nodes need to stake token, once cheating behaviors are discovered, penalties will be given out to terminal nodes.
